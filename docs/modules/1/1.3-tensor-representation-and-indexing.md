# Module 1.3 — Tensor Representation and Indexing

> *A tensor becomes useful only when a runtime can map logical indices to physical memory.*

---

# Objective

By the end of this module, you should be able to:

* derive contiguous strides automatically;
* enforce fundamental tensor invariants;
* compute storage offsets from logical indices;
* implement scalar tensor indexing.

# Pure Algorithms

Rather than embedding all logic inside `Tensor`, several algorithms are implemented independently.

Examples include:

* `compute_tensor_size()`
* `compute_contiguous_strides()`
* `compute_offset()`

These algorithms:

* are deterministic;
* are independently testable;
* do not depend on tensor state.

The tensor coordinates them but does not implement them.

---

# Tensor Construction

During initialization the runtime:

1. verifies storage capacity;
2. derives contiguous strides when omitted;
3. validates rank consistency.

After construction, every tensor satisfies the implemented invariants.

---

# Tensor Invariants

The implementation guarantees:

* storage capacity is sufficient for the tensor shape;
* shape and strides always describe the same rank;
* strides are always available after construction.

These guarantees simplify every future tensor operation.

---

# Scalar Indexing

The first supported indexing operation is:

```python
tensor[i, j]
```

The runtime performs the following steps:

1. validate index rank;
2. validate index bounds;
3. compute the storage offset;
4. access storage.

The offset is computed as:

```
offset = Σ(index × stride)
```

This algorithm works for tensors of arbitrary rank.

---

# Design Decisions

Several implementation decisions intentionally simplify the runtime.

Current implementation does not yet support:

* slicing;
* tensor views;
* transpose;
* reshape;
* storage offsets.

These features require additional metadata and will be introduced later.

---

# Engineering Insight

The tensor itself contains almost no algorithms.

Instead, it coordinates specialized components responsible for:

* shape computations;
* stride computations;
* indexing;
* storage access.

This separation improves readability, testability, and future extensibility.

# Connection to the AI Stack

Modern tensor runtimes perform exactly the same conceptual process.

Whether the backend is NumPy, PyTorch, ONNX Runtime or TensorRT, every runtime must eventually translate logical indices into storage offsets.

The implementation developed in this module captures this fundamental idea in its simplest form.
