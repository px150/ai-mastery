# Artificial Intelligence Is an Ecosystem

A common misconception is to think of systems such as ChatGPT as "a model."

In reality, modern AI applications are ecosystems composed of many interacting components.

The model is only one part of a much larger architecture.

Understanding the interactions between these components is often more important than understanding each component individually.

---

Artificial Intelligence is a broad discipline that includes many different approaches.

Modern language models represent only a small branch of this hierarchy.


# Training vs Inference

One of the most important distinctions in modern AI systems is the difference between training and inference.

## Training

Training is the process of learning model parameters from data.

Its objective is to produce a trained model.

Typical components include:

* tensors;
* automatic differentiation;
* optimizers;
* loss functions.

Training is computationally expensive and performed relatively infrequently.

---

## Inference

Inference is the process of using a trained model to generate predictions.

Typical components include:

* runtime;
* scheduling;
* batching;
* memory management;
* KV cache;
* token generation.

Inference happens continuously in production systems and is often optimized for latency, throughput, or both.

---

# Model vs Runtime

A trained model is not executable by itself.

The model defines *what* should be computed.

The runtime determines *how* it is executed efficiently on the available hardware.

This distinction is fundamental throughout AI Mastery.

---

# Major Open Source Projects

The AI ecosystem consists of specialized projects with different responsibilities.

| Project      | Primary Responsibility                     |
| ------------ | ------------------------------------------ |
| PyTorch      | Training framework                         |
| NumPy        | Numerical computing                        |
| ONNX         | Framework-independent model representation |
| ONNX Runtime | Generic inference runtime                  |
| TensorRT     | Optimized NVIDIA inference                 |
| llama.cpp    | Efficient local inference                  |
| vLLM         | Large-scale inference serving              |
| MLIR         | Compiler infrastructure                    |
| CUDA         | GPU programming platform                   |
| cuDNN        | GPU deep learning kernels                  |

Learning these responsibilities is more important than memorizing implementation details.

# Key Takeaways

After completing this module, you should understand that:
